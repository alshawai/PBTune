# Visualization Guide

This document explains the figures generated by the AI Database Optimization framework.
The visualization module (`src.visualization`) produces publication-quality charts to compare
the performance, convergence, efficiency, and tradeoffs of different database tuning algorithms
(e.g., PBTune, BO-SMAC, Default).

All outputs are styled via `PBTuneTheme` for specific publication venues (`pvldb`, `springer`,
`preview`) to ensure consistent fonts, colorblind-friendly palettes, and correct column dimensions.
The framework defaults to a **Boxed Academic Style** (all spines visible, top/right legends, hatched bars) matching CDBTune+ and other top-tier systems papers.

---

## Quick Start

```bash
# Minimal: just PBT and BO directories
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt \
    --bo results/sysbench/bo

# Full: with comparison report, custom metric, and venue
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt \
    --bo results/sysbench/bo \
    --comparison-path results/sysbench/comparison/report.json \
    --metric latency_p95 \
    --venue pvldb \
    --output-dir analysis_final
```

### Global CLI Arguments

To generate all plots across a variety of metrics easily, we provide a standalone script:
```bash
python generate_all_metrics.py
```
This automatically loops through `score`, `throughput`, `latency_p99`, and `latency_p95` and exports suffixed variations.

If you are generating plots individually using the `__main__` entrypoint:

| Argument | Required | Default | Description |
|---|---|---|---|
| `--pbt` | Yes | — | Path(s) to PBT result directories or JSON files |
| `--bo` | Yes | — | Path(s) to BO result directories or JSON files |
| `--comparison-path` | No | `None` | Path to a multi-arm comparison JSON report |
| `--output-dir` | No | `analysis_final` | Directory to save generated figures |
| `--venue` | No | `preview` | Target venue: `pvldb`, `springer`, or `preview` |
| `--metric` | No | `score` | Y-axis metric for convergence curve (see below) |

> **Note**: Paths can be either directories (e.g. `results/sysbench/pbt/`) or individual JSON files.
> When a directory is given, all matching `pbt_results_*.json` or `bo_results_*.json` files
> are loaded automatically.

## 1. Convergence Curve (`convergence_curve.py`)

### Purpose

Shows how optimization algorithms improve over time. Demonstrates convergence speed and
sample efficiency of PBT vs BO.

### What it plots

| Panel | X-axis | Y-axis | Shows |
|---|---|---|---|
| **Left** | Cumulative Evaluations | Selected metric | Sample efficiency |
| **Right** | Wall-Clock Time (s) | Selected metric | Real-world speed |

A horizontal dashed line shows the **Default** baseline configuration.
When multiple seeds are loaded, shaded bands show ±1 std deviation.

### Data sources

| Source | How it's used |
|---|---|
| `--pbt` (required) | Reads full `generation_history` → per-generation worker scores/metrics → running-best curve |
| `--bo` (required) | Reads full `evaluation_history` → per-iteration scores/metrics → running-best curve |
| `--comparison-path` (optional) | Only reads the `"default"` arm to draw the baseline horizontal line. Falls back to PBT's first generation if missing |

### `--metric` argument

Controls what value is plotted on the Y-axis:

| Value | Y-axis label | Direction | Description |
|---|---|---|---|
| `score` (default) | Composite Score | Higher is better | Weighted composite from the scoring engine |
| `latency_p95` | Latency P95 (ms) | Lower is better | Raw 95th percentile latency |
| `latency_p99` | Latency P99 (ms) | Lower is better | Raw 99th percentile latency |
| `throughput` | Throughput (TPS) | Higher is better | Raw transactions per second |

> For latency metrics, the "best" curve uses `minimum.accumulate` instead of `maximum.accumulate`.

### Examples

```bash
# Default: composite score
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo --metric score

# Compare by P95 latency
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo --metric latency_p95

# Compare by throughput with Default baseline from comparison
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo \
    --comparison-path report.json \
    --metric throughput
```

### Programmatic API

```python
from src.visualization.plots.convergence_curve import generate

fig = generate(
    pbt_paths=["results/sysbench/pbt"],
    bo_paths=["results/sysbench/bo"],
    comparison_path="report.json",  # optional
    metric_key="latency_p95",       # or None, "throughput", "latency_p99"
    venue="preview",
    formats=["pdf", "png"],
)
```

## 2. Pareto Frontier (`pareto_frontier.py`)

### Purpose

Visualizes the multi-objective tradeoff between **throughput** (higher is better) and
**P95 latency** (lower is better) for each method's best configurations.

### What it plots

A scatter plot with:
- X-axis: Throughput (Queries/sec)
- Y-axis: 95th Percentile Latency (ms)
- Each point = one seed/run's best configuration
- Color-coded by method (PBTune, BO-SMAC, Default)

### Data sources

The plot supports **two modes** — use either or both:

| Source | What it reads | Methods shown |
|---|---|---|
| `--pbt` + `--bo` files | `best_configuration.metrics` → `throughput` and `latency_p95` per seed | PBTune, BO-SMAC |
| `--comparison-path` | `runs_by_arm.*.metrics` → `throughput` and `latency_p95` per run | Default, BO-SMAC, PBTune (all arms) |

> **Priority**: When `--comparison-path` is provided and has data, it takes priority.
> PBT/BO files are used as fallback when comparison is empty or missing.

> **Tip**: Use `--comparison-path` to get the **Default** arm on the scatter plot.
> Without it, only PBTune and BO-SMAC are shown.

### Examples

```bash
# From PBT/BO files only (2 methods)
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo

# From comparison report (3 methods including Default)
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo \
    --comparison-path report.json
```

### Programmatic API

```python
from src.visualization.plots.pareto_frontier import generate

# From comparison only
fig = generate(comparison_path="report.json", venue="preview")

# From PBT/BO files only
fig = generate(pbt_paths=["results/sysbench/pbt"], bo_paths=["results/sysbench/bo"])

# Both (comparison takes priority)
fig = generate(
    pbt_paths=["results/sysbench/pbt"],
    bo_paths=["results/sysbench/bo"],
    comparison_path="report.json",
)
```

## 3. Resource Efficiency (`resource_efficiency.py`)

### Purpose

Compares the **memory utilization** of the best configurations found by each method.
Shows whether aggressive tuning comes at the cost of excessive resource consumption.

### What it plots

A boxplot + strip plot showing:
- X-axis: Method (PBTune, BO-SMAC)
- Y-axis: Memory Utilization (bytes)
- Each point = one seed's best configuration

### Data sources

| Source | What it reads |
|---|---|
| `--pbt` (required) | `best_configuration.metrics.memory_utilization` per seed |
| `--bo` (required) | `best_configuration.metrics.memory_utilization` per seed |
| `--comparison-path` | **Not used** (parameter accepted but ignored) |

### Examples

```bash
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo
```

### Programmatic API

```python
from src.visualization.plots.resource_efficiency import generate

fig = generate(
    pbt_paths=["results/sysbench/pbt"],
    bo_paths=["results/sysbench/bo"],
    venue="preview",
)
```

## 4. Comparison Bar Chart (`comparison_bar.py`)

### Purpose

A grouped bar chart specifically comparing multiple methods across multiple metrics (e.g., Throughput and Latency) simultaneously. This replaces multi-panel scatter plots when strict quantitative comparisons are required.

### What it plots

A multi-panel figure where each panel corresponds to a metric:
- X-axis: Different database workloads (e.g., Read-Only, Read-Write, Write-Only)
- Y-axis: The metric value (e.g., Throughput or Latency)
- Bars: Grouped by method, stylized with explicit hatches and bordered legends spanning the top of the figure.

### Data sources

| Source | What it reads |
|---|---|
| `extensive/comparisons/` | `multi_arm_comparison_*.json` files for workload summaries |

### Examples

```bash
python -m src.scripts.generate_final_visuals \
    --figure comparison_bar \
    --data-dir results/oltp
```

## 5. Performance Distribution (`performance_distribution.py`)

### Purpose

Shows the **statistical distribution** of final best scores across seeds for each method.
Includes automated significance testing (Mann-Whitney U) with bracket annotations.

### What it plots

A violin + strip plot (or Raincloud if `ptitprince` is installed) showing:
- X-axis: Method (Default, BO-SMAC, PBTune)
- Y-axis: Final Best Composite Score
- Significance brackets: `*` (p<0.05), `**` (p<0.01), `***` (p<0.001), `ns`

### Data sources

| Source | What it reads | Purpose |
|---|---|---|
| `--pbt` (required) | Full history → `best_scores[-1]` per seed | PBTune final scores |
| `--bo` (required) | Full history → `best_scores[-1]` per seed | BO-SMAC final scores |
| `--comparison-path` (optional) | All arms (`default`, `bo`, `pbt`) rescored with shared normalization | More accurate scores + **Default** arm. Without it, Default is not shown |

> **Note**: When `--comparison-path` is provided, its rescored values **replace** the
> raw PBT/BO scores to ensure all arms use the same normalization. This is the
> recommended mode for publication figures.

### Examples

```bash
# Without comparison (2 methods, no Default, no rescoring)
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo

# With comparison (3 methods, rescored, significance tests)
python -m src.scripts.generate_final_visuals \
    --pbt results/sysbench/pbt --bo results/sysbench/bo \
    --comparison-path report.json
```

### Programmatic API

```python
from src.visualization.plots.performance_distribution import generate

fig = generate(
    pbt_paths=["results/sysbench/pbt"],
    bo_paths=["results/sysbench/bo"],
    comparison_path="report.json",  # optional, adds Default arm
    metric_key="throughput",        # dynamically changes the plotted metric
    show_significance=False,        # disabled by default for cleaner plots
    venue="preview",
)
```

## Data Sources Summary

```
                        --pbt        --bo         --comparison-path       --metric
Convergence Curve       FULL hist    FULL hist    optional (Default)       ✅ yes
Pareto Frontier         BEST cfg     BEST cfg     optional (all arms)      ❌ no
Resource Efficiency     BEST cfg     BEST cfg     ❌ ignored               ❌ no
Perf Distribution       FINAL best   FINAL best   optional (all arms)      ✅ yes
Comparison Bar          ❌ ignored   ❌ ignored   REQUIRED                 ❌ no
```

- **FULL hist** = reads entire `generation_history` / `evaluation_history`
- **BEST cfg** = reads only `best_configuration.metrics`
- **FINAL best** = reads history but only uses `best_scores[-1]`

## Customizing and Exporting

### Venue Presets

| Venue | Font | Width | LaTeX | Use case |
|---|---|---|---|---|
| `preview` | Sans-serif 11pt | 10" double | No | Quick local preview / fallback if LaTeX fails |
| `pvldb` | Serif 9pt | 7" double | Yes | VLDB submission |
| `springer` | Serif 10pt | 6.85" double | Yes | Springer/LNCS submission |

> **LaTeX Note**: `pvldb` and `springer` venues require a working LaTeX installation (specifically `type1cm.sty` and `texlive-fonts-extra`). If Matplotlib fails with a `RuntimeError: latex was not able to process`, fallback to `--venue preview` or install the missing TeX packages.

### Output Formats

All figures are exported as both `.pdf` (vector) and `.png` (raster, 300 DPI) by default.
Override with the `formats` parameter in the programmatic API.

### Theme and Colors

- Consistent method colors and explicit hatching (`METHOD_HATCHES`) via `src.visualization.colors`
- Dense, fully-boxed academic styling (no despining) matching reference literature
- Grid, spine, and typography settings via `PBTuneTheme`
